花畑ゲーム 仕様書

1. プロジェクト概要

1.1 基本情報

  • プロジェクト名: Territory Battle Game
  • ゲーム名 Try Flower
  • 開発環境: Unity 6.1 + C#
  • ジャンル: リアルタイム戦略ゲーム
  • プラットフォーム: PC（Windows/Mac/Linux 対応）
  • 開発期間: 予定開発期間に応じて調整

1.2 ゲーム概要

5 人対戦の領地拡張ゲーム。プレイヤー 1 人 + CPU4 人で、制限時間内に花を植えて領地を拡張し、最も多くの領地を獲得したプレイヤーが勝利する。

2. ゲームシステム

2.1 基本ルール

  • プレイヤー数: 5 人（人間 1 人 + CPU4 人）
  • ゲーム時間: 5 分間
  • 勝利条件: ゲーム終了時点で最も多くの領地を所持しているプレイヤーの勝利
  • 視点: 斜め上から見下ろす 3D ビュー（クォータービュー）

2.2 マップ仕様

  • サイズ: 30×30 セル（900 セル）
  • セルサイズ: 1m×1m
  • スタート地点: 5 箇所（四隅 + 中央）
  • 地形: 庭園ベース
  • 障害物: 岩、低木などがランダム配置
  • 侵入不可エリア: 池などがランダム配置
  • セルの境界線: セルの境界はわかりやすく薄い線を表示する

2.3 花・領地システム

2.3.1 基本的な花植えルール

  • キャラクターは任意のセルに花を植えることができる
  • 花の種類はキャラクターごとに異なる見た目とする
  • 花は植えた瞬間に咲く（花のアセットを足元に配置する）
  • 既に花が植えられたセルにも追加で花を植え可能
  • 最も多くの花を植えたプレイヤーがそのセルの所有者となる
  • セルの所有者が決まっているセルはセルの境界がキャラクターと同じ色に輝くようにし、セルにキャラクターと同じ色の輝くエフェクトを配置する

2.3.2 囲み判定（囲碁ルール適用）

  • 同じ所有者のセルで完全に囲まれたセルは、花が植えられていなくてもその所有者の領地となる
  • 囲まれたセル内には他のプレイヤーは花を植えることができない
  • 領地の所有者が変わった場合、囲み判定を再計算し、囲いが成立しなくなったセルは解放される
  • 侵入負荷のセル（画面端、岩などのオブジェクト）は全プレイヤーの囲み判定用セルとして使用される

2.4 移動システム

  • 操作方法: WASD キーでキャラクター移動
  • カメラ操作: マウスでカメラ移動、マウスはキャラクターの後ろから見下ろす位置で追従する三人称視点
  • 移動制限: マップ内を自由に移動可能
  • 障害物: 岩、低木、池などは通行不可
  • 移動速度: 基本移動速度（アイテムで変更可能）

2.5 アクションシステム

  • 花植え: スペースキーで足元のセルに花を植える
  • アイテム使用: 左 Shift キーでアイテムを使用
  • 移動: WASD キーで 4 方向移動（前後左右）
  • 方向転換: マウスでキャラクターの向きを変更

3. アイテムシステム

3.1 木の葉アイテム概要

  • 空からランダムに降ってくる
  • ゲーム開始時には 5 ～ 8 個のアイテムがランダムに配置されている
  • プレイヤーが触れることで自動取得
  • アイテムは 1 つしか持つことができず、アイテムを持った状態で新しいアイテムを取得すると、古いものは破棄される
  • 8 種類の異なる効果を持つ
  • 各アイテムには使用回数制限がある

3.2 アイテム詳細仕様

3.2.1 タイプ A（スコップマーク）

  • 効果: 足元セルの花全てを別のセルに移植
  • 使用回数: 5 回
  • 使用方法: アイテム使用後、移植先のセルをクリック

3.2.2 タイプ B（時計マーク）

  • 効果: 10 秒後に咲く遅延花を植える + 移動速度低下効果
  • 使用回数: 10 回
  • 詳細: 他プレイヤーがそのセルを通過時に移動速度が低下

3.2.3 タイプ C（ジョウロマーク）

  • 効果: 自分の周囲 3×3 セルに一度に花を植える
  • 使用回数: 3 回
  • 範囲: キャラクターを中心とした 3×3 の 9 セル

3.2.4 タイプ D（キラキラマーク）

  • 効果: 15 秒間、確率で 3 倍カウントの輝く花を植えられる
  • 使用回数: 1 回
  • 詳細: 通常の花の 3 倍の価値を持つ特殊な花で、発生確率は 40%

3.2.5 タイプ E（靴マーク）

  • 効果: 20 秒間移動速度アップ + 速度低下無効化
  • 使用回数: 1 回
  • 詳細: 効果時間中は他のプレイヤーの速度低下効果を受けない

3.2.6 タイプ F（岩マーク）

  • 効果: 岩の配置
  • 使用回数: 2 回
  • 詳細: 前方のセルに侵入不能オブジェクトを配置する

3.2.7 タイプ G（鍬マーク）

  • 効果: 足元から前方 5 セルに花を植える
  • 使用回数: 3 回
  • 範囲: キャラクターの足元を起点として前方 5 セル

3.2.8 タイプ H（！マーク）

  • 効果: キャラクターの位置入れ替え
  • 使用回数: 1 回
  • 詳細: 全キャラクターの位置をランダムで入れ替える

4. UI 仕様

4.1 ゲーム画面 UI

  • 残り時間表示: 画面上部中央に大きく表示
  • 領地カウント: 各プレイヤーの現在の領地数を表示
  • 順位表示: リアルタイムでの順位を表示
  • アイテムスロット: 所持アイテムと使用回数を表示
  • アイテム硬化時間: アイテム使用後に効果が継続するものは、残り時間をキャラクターの上部にバーで表示
  • 花カウント表示: セルごとの花の本数を数字で表示
  • 一時停止ボタン: ゲーム中断機能

4.2 画面構成

4.2.1 タイトル画面

  • ゲームタイトル
  • スタートボタン
  • 設定ボタン
  • 終了ボタン

4.2.2 キャラクター選択画面

  • 5 体のキャラクターから 1 体を選択
  • キャラクターの 3D プレビュー
  • 決定ボタン
  • 戻るボタン

4.2.3 ゲーム画面

  • 3D ゲームフィールド
  • UI 要素（上記 4.1 参照）
  • ミニマップ（オプション）

4.2.4 リザルト画面

  • 最終順位表示
  • 各プレイヤーの領地数
  • 再戦ボタン
  • キャラクター選択に戻るボタン
  • タイトルに戻るボタン

4.3 一時停止機能

  • 呼び出し: ESC キーまたは一時停止ボタン
  • 機能: ゲーム続行、設定、タイトルに戻る

5. 技術仕様

5.1 開発環境

  • エンジン: Unity 6.1
  • 言語: C#
  • 対象プラットフォーム: Windows, Mac, Linux

5.2 必要アセット

  • キャラクター: Unity アセットストアから 5 体の異なるキャラクター
  • マップ素材: 庭園風の地形、岩、低木、池など
  • 花素材: 複数色対応の花モデル
  • UI 素材: アイコン、ボタン類
  • エフェクト: パーティクル効果、アニメーション

5.3 システム構成

5.3.1 主要クラス設計

GameManager: ゲーム全体の管理
SceneManager: シーン遷移管理
GridManager: グリッドシステム管理
PlayerController: プレイヤー操作制御
CPUController: CPU行動制御
CellData: セル情報管理
TerritorySystem: 領地判定システム
ItemSystem: アイテム管理
UIManager: UI制御

5.3.2 データ管理

  • セル状態データ（所有者、花の数、特殊状態）
  • プレイヤーデータ（位置、所持アイテム、領地数）
  • ゲーム状態データ（残り時間、順位、一時停止状態）

5.4 パフォーマンス要件

  • フレームレート: 60fps 安定動作
  • メモリ使用量: 最大 2GB 以下
  • 読み込み時間: シーン遷移 3 秒以内

6. AI 仕様

6.1 CPU 行動パターン

  • 基本行動: ランダム移動 + 空きセルへの花植え
  • 戦略行動: 他プレイヤー領地への侵入
  • 防御行動: 自分の領地周辺の強化
  • アイテム使用: 簡単な優先度判定による使用

6.2 難易度調整

  • 思考時間: CPU 行動の間隔調整
  • 戦略性: ランダム行動と戦略行動の比率
  • アイテム使用頻度: アイテム使用の積極性

7.2 音量設定
